01 项目概述

本项目实现了一个可运行的 Atoms-like Demo，命名为 BuilderOS：用户通过自然语言描述需求，平台使用 LLM 网关和多智能体流程生成一个可交互网页应用，同时产出项目文件夹、源码包、版本记录和发布预览。作品目标不是做一个静态仿站，而是交付一个评委可以真实注册、登录、构建、查看源码、下载产物和验证服务端数据的 AI Native 应用生成平台原型。

产品定位：Atoms 偏"输入想法后自动生成产品"，BuilderOS 在这个基础上额外强调过程可见、Agent 可配置、执行可审查。本 Demo 除了实现 Atoms-like 的首页生成体验，还吸收了另一套 OmniAgent 平台中的 Agent 编排思路，把隐式多智能体工作流做成显式自由编排能力。
补充说明：OmniAgent 是本人自研的另一套 Agent / RAG 平台，方向与 BuilderOS 相似。本 Demo 借鉴其"模型路由、知识库 grounding、Agent 编排"的产品思路。感兴趣可访问：agent.poppcic.cn

相对 Atoms，BuilderOS 重点扩展了三类能力：

  • 显式 Agent 编排：把自动化 Agent 链路变成可配置、可审查的工作流
  • 知识 grounding：把知识库、文件解析和 RAG 证据作为生成结果的依据
  • 运行审计与可观测性：把 API 健康、服务状态、持久化路径、运行记录和 Agent Orchestrator 状态通过实时数据页暴露出来

03 重点亮点：流程编排与 Agent 调度

在线演示入口：builder.poppcic.cn/?section=orchestration
登录演示账号后进入「编排」页，点击右上角「试运行链路」或左下角「开始执行」，即可触发后端 /api/orchestrations/:id/run 串行调度。

5.1 流程编排

BuilderOS 在 Atoms-like 应用生成主流程之外，增加了一个 OmniAgent 风格的显式 Agent 编排能力。每个步骤包含：Agent 名称、角色职责、系统提示词、模型路由、执行动作、输出产物、工具绑定、守护规则。

5.2 Agent 调度

采用串行调度模型：总控 → 需求分析 → 简历检索 → 匹配评分 → 面试题 → 报告汇总。服务端每个 Agent 节点按"节点模型配置优先，其次继承顶部路由"规则调用 LLM。本轮线上验证中，链路使用 qwen-plus 完成 6 个节点调用，命中 9 条知识库证据，输出来自服务端知识库的候选人排序。

5.3 流程可视化

编排页使用三栏执行工作台：

  • 左侧「任务说明」：任务背景、适用场景、编排要求和当前链路
  • 中间「任务流程」：垂直节点展示 Agent 执行链路，区分待执行/执行中/已完成
  • 右侧「任务结果」：真实 Agent 输出摘要、LLM provider、知识库证据数量、候选人排序和最终交付

5.4 校验与可审查性

  • 配置写入服务端 state.json，刷新页面后仍可恢复
  • /api/status 中新增 Agent Orchestrator 服务项
  • /api/cloud-resources 中新增「Agent 自由编排」资源卡，展示 flows / steps 数量
  • 试运行生成 Execution Trace，记录每个 Agent 的动作、耗时、LLM provider、证据标题和输出
  • 每个步骤含 guardrail：输出必须附带依据、分数必须可解释、结论和证据一一对应

04 实现思路与关键取舍

  • React + TypeScript + Vite 快速构建可体验产品原型
  • 首次进入提供工作区初始化和注册，用户信息、密码哈希和会话写入 MySQL
  • 登录不会自动创建用户；侧边栏提供退出登录入口，退出时清除本地 token 并调用服务端 logout 让 session 失效
  • 服务端支持 OpenAI 兼容 /chat/completions，密钥通过环境变量配置并在状态接口中脱敏展示
  • 登录后顶部提供模型路由选择器：Auto / qwen-plus / gpt-5.5；选择 gpt-5.5 时直接走 token-qiv.cn 中转站；当模型不可用时自动降级到本地模板生成器
  • 积分当前只作为额度展示，不限制构建次数——避免评审被 quota 阻断
  • 根据 prompt 识别 SaaS / 电商 / 招聘 / 研究 / 视频等类型，生成对应 React/Vite 项目文件
  • 使用 iframe srcDoc 渲染生成结果，让评审可直接操作生成出的应用
  • 服务端提供 zip 源码包下载接口，评审可直接下载完整生成项目和 manifest
  • 生产环境将生成文件落盘到 /opt/builderos/data/generated-projects/project-<id>/
  • Race Mode 作为延展能力：开启后生成多个方案方向和评分，体现多模型竞争思路

07 后续扩展优先级

P1
沙箱执行：接入隔离容器，真实运行 npm install + npm run build + 自动修错循环，让平台从「生成代码」升级为「生成并验证代码」
P2
显式自由编排增强：把当前串行 Agent 链升级为可拖拽 DAG，补充分支条件、失败重试、人工确认、节点级超时和审计日志
P3
RAG 信息架构升级：先支持多知识库/每库多资料/构建时选择召回范围，再升级 embeddings + 向量数据库 + rerank，补充 PDF/DOCX 解析
P4
GitHub OAuth + 发布流水线：平台内一键创建仓库、提交生成文件、打开 PR，再接入 Vercel/Cloudflare Pages API 实现真实部署
P5
计费与积分系统：模型调用按 provider/model 记录 token + 成本；充值写入 balance ledger，消费扣减；订阅套餐提供团队席位、私有项目和并发构建
P6
多模型调度 + 团队协作：按任务类型选择「便宜/强/快」模型；增加 workspace 成员、角色权限、版本 diff 和审查记录
